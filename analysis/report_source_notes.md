## Validation notes

- Independent SQLite union query confirms 156 represented departments, no orphan
  person-to-org or org-to-parent links, and a maximum of exactly 25 people per org.
- Counts from the independent query can drift upward by a few rows because
  `rest-batch` remains active; the report uses the frozen aggregate JSON timestamped
  above.
- The three stored crawl errors are Windows console encoding errors. In
  `engine.py`, progress logging occurs after the page transaction commits but
  inside the same `try` block, so a logging failure can relabel an already-processed
  page as an error.
- The notebook is structurally valid JSON but could not be executed because the
  environment lacks `nbformat`, `nbclient`, and Jupyter. Exact execution command
  after installing those dependencies:
  `py -m jupyter nbconvert --execute --to notebook --inplace analysis/geds_analysis.ipynb`

## Caveats

- Person and title counts are lower bounds until GEDS person-list pagination is implemented.
- The active remainder batch makes 26 department totals partial.
- Keyword categories overlap and are heuristic; they are discovery signals, not workforce classifications.
- Same-name and same-signature collisions are review candidates, not confirmed duplicate people.
